Atomic Transaction

Mọi thao tác chỉnh sửa đơn hàng được thực hiện trong một atomic transaction. Nếu bất kỳ bước nào thất bại, toàn bộ thay đổi sẽ được rollback — không bao giờ xảy ra tính phí trùng, lỗi thuế, hoặc mất tồn kho. Đây là điểm khác biệt lớn so với đối thủ.

05

Lộ Trình Phát Triển

Từ MVP đến Growth features

P0 — MVP Kế Hoạch 4 Tuần Chi Tiết

Tuần Mục Tiêu Chi Tiết Công Việc
Tuần 1 Project setup & nền tảng Shopify app config, Firebase setup, Firestore schema (shops, orders, editSettings, orderEdits), webhook endpoints (orders/create, orders/updated, orders/fulfilled, app/uninstalled), theme extension skeleton trên Order Status Page
Tuần 2 Chỉnh sửa đơn hàng (core) Backend API: edit eligibility check, đổi địa chỉ, đổi variant (orderEditBegin → addVariant → commit), điều chỉnh số lượng (setQuantity). Storefront UI: form chỉnh sửa trên Order Status Page, hiển thị chênh lệch giá
Tuần 3 Hủy đơn & thanh toán Hủy đơn hàng + time window (Cloud Tasks tự động hết hạn), tự động hoàn tiền / gửi invoice khi giá thay đổi, quản lý tồn kho (restock khi edit/cancel), kiểm tra inventory trước khi cho phép swap
Tuần 4 Admin UI & hoàn thiện Merchant editing UI trong admin (React/Polaris), trang cấu hình editSettings, email thông báo sau khi edit/cancel, testing toàn bộ flow, fix bug, submit Shopify App Store review
