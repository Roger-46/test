Countdown Timer — Display Rules

Quy tắc hiển thị thời gian còn lại cho phép chỉnh sửa đơn hàng trên storefront widget.

1. Quy tắc format thời gian

Hiển thị tối đa 2 đơn vị thời gian lớn nhất, làm tròn xuống. Không hiển thị giây.

Thời gian còn lại Format Ví dụ Ghi chú
>= 7 ngày{W}w {D}d2w 3d, 1w 1dNếu D = 0 → chỉ "2w"
1 — 6 ngày{D}d {H}h5d 12h, 2d 3hNếu H = 0 → chỉ "2d"
1 — 23 giờ{H}h {M}m23h 45m, 1h 30mNếu M = 0 → chỉ "2h"
1 — 59 phút{M}m45m, 10mKhông hiện giây
< 1 phút<1m<1mHiện "<1m" thay vì giây

5. Edge cases
Tình huống Hiển thị Hành vi
Mở trang khi timer = 0Widget ẩnWidget ẩn hoàn toàn, không cho edit
Timer hết khi đang edit⏱ <1m remainingCho hoàn thành edit đang mở, không cho bắt đầu edit mới
Timer hết sau khi submit—Edit đã submit vẫn được xử lý (grace period)
Order đã fulfilledWidget ẩnKhông hiện timer, không hiện widget
Order đã cancelledWidget ẩnKhông hiện timer
Merchant đổi time window—Order cũ giữ nguyên. Chỉ order mới áp dụng setting mới.
Tab bị ẩn / browser minimize—Khi quay lại, sync lại từ server. Không dựa vào client timer.
6. Technical notes
Update interval: Timer cập nhật mỗi 60 giây. Sync lại từ server khi tab được focus trở lại (visibilitychange event).
Settings: Time window options: 30m / 1h / 2h / 4h / 12h / 24h / 2d / 3d / 7d / 14d / 30d / Until fulfillment
