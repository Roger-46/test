Khi đẩy PRD lên Notion task, PHẢI theo cấu trúc 9 mục này:

1. **Context** (User Stories) — Problem, Business Value, User Stories
2. **Business Solutions** — Scope IN/OUT, what we're solving
3. **Product Solutions** — Features, flow, how we solve it
4. **Acceptance Criteria** — AC từ User Stories + Edge Cases
5. **Technical Notes** — Data Model, API Design, Architecture, Resolution Logic, Security
6. **Release Plan** — Implementation Breakdown (Backend + Frontend, NO testing), Risks
7. **Giao diện** — ĐỂ TRỐNG (Roger tự điền)
8. **UI Specifications** — UI layout, states, defaults
9. **Test case** — ĐỂ TRỐNG (Roger tự điền)

**Why:** Roger muốn format thống nhất giữa các task Notion, dễ export PDF. Mục 7 và 9 luôn để trống.

**How to apply:** Khi viết script đẩy PRD lên Notion, map nội dung PRD vào 9 mục trên. Bắt đầu bằng "Mục Lục" liệt kê 9 mục.
