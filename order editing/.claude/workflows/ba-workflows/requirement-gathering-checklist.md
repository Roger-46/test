# Requirement Gathering Checklist

Use this checklist when gathering requirements for new features.

## ✅ Pre-Gathering Preparation

- [ ] Define feature name & scope
- [ ] Identify stakeholders (merchants, users, team)
- [ ] Gather any existing documents/feedback
- [ ] Schedule time with stakeholders
- [ ] Prepare question list

## ✅ During Gathering

### Understanding the Problem
- [ ] What problem are we solving?
- [ ] Who has the problem? (user type)
- [ ] Why is it important? (business impact)
- [ ] How urgent is this? (timeline)

### Understanding the Solution
- [ ] What should the feature do?
- [ ] Who uses it? (user type, count)
- [ ] When would they use it? (workflow)
- [ ] Are there alternative approaches?

### Understanding Constraints
- [ ] Team capacity (how many devs available?)
- [ ] Technical constraints (API limits, architecture)?
- [ ] Timeline (when needed?)
- [ ] Budget/resources (what's available)?
- [ ] Shopify limitations (API, app constraints)?

### Understanding Success
- [ ] How do we measure success?
- [ ] What's the acceptance criteria?
- [ ] Performance requirements?
- [ ] User experience expectations?

## ✅ After Gathering

### Documentation
- [ ] Write user story/requirement doc
- [ ] Document acceptance criteria
- [ ] List dependencies & blockers
- [ ] Add to requirements folder (@docs/requirements/)

### Validation
- [ ] Share with stakeholders for confirmation
- [ ] Share with devs for feasibility check
- [ ] Share with tester for test planning
- [ ] Get sign-off from decision maker

### Planning
- [ ] Estimate effort (dev + tester)
- [ ] Assign priority (P0/P1/P2)
- [ ] Identify risks & blockers
- [ ] Schedule for sprint
- [ ] Log decision in decision-log.md

## Questions to Ask Stakeholders

### For Merchants (Users)
1. "What problem are you facing now?"
2. "How much time does this problem take?"
3. "What's your current workaround?"
4. "How often would you use this?"
5. "What would be success to you?"

### For Business/PM
1. "Why is this important to the business?"
2. "What's the revenue impact?"
3. "When do we need this?"
4. "Who requested this feature?"
5. "Is this a blocker for other work?"

### For Dev Team
1. "Is this technically feasible?"
2. "What's the effort estimate?"
3. "Are there technical constraints?"
4. "Do we need new infrastructure?"
5. "Can this be done in parallel?"

### For QA/Tester
1. "What scenarios should we test?"
2. "Are there edge cases?"
3. "What could break?"
4. "Performance requirements?"
5. "Security concerns?"

## BA Workflow Template

```
1. Stakeholder conversation
   ↓
2. Write user story/requirement
   ↓
3. Share with team for feedback
   ↓
4. Refine based on feedback
   ↓
5. Get sign-off from stakeholders
   ↓
6. Move to "Ready for Dev" status
   ↓
7. Share with dev team
   ↓
8. Log decision in decision-log.md
```
