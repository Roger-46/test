MVP Core — Foundation + Backend Tuần 1-4 • P0

Xây dựng nền tảng: Shopify OAuth, Firestore schema, order sync, edit engine, storefront widget

1.1
Shopify OAuth + App Setup
ICE 800
Nó là gì: Shopify OAuth flow hoàn chỉnh, session token auth, HMAC webhook verification, app bridge setup. Firebase project init, Firestore security rules, hosting config.

Để làm gì: Đây là foundation cho mọi thứ. Không có OAuth = không có app. Không có Firestore = không lưu data. Mọi feature khác đều phụ thuộc vào milestone này.

Trông như thế nào: Merchant click "Install" trên App Store → OAuth flow → redirect về embedded app → dashboard trống (chưa có data). Backend: Firebase Functions chạy, Firestore có collection shops với shop data cơ bản.
BackendInfra
Chưa làm
1.2
Order Sync + Webhook Listeners
ICE 630
Nó là gì: Đồng bộ đơn hàng từ Shopify về Firestore qua webhooks (orders/create, orders/updated, orders/cancelled). Lưu order data cần thiết: line items, shipping address, customer info, fulfillment status. Real-time sync khi có thay đổi.

Để làm gì: App cần data đơn hàng để biết đơn nào có thể edit. Webhook là cách duy nhất để nhận thông báo real-time từ Shopify. Không có order sync = app không biết gì về đơn hàng của merchant.

Trông như thế nào: Merchant cài app → orders tự động sync về Firestore. Khi có đơn mới hoặc update → webhook fire → Firestore cập nhật. Admin UI hiện danh sách orders với status mới nhất.

1.3
Edit Window + Eligibility Engine
ICE 648
Nó là gì: Logic xác định đơn hàng nào đủ điều kiện để edit. Cấu hình edit time window (30 phút → until fulfillment). Rules: chỉ cho edit khi chưa fulfilled, trong time window, không bị cancelled. Merchant tuỳ chỉnh window per product/collection.

Để làm gì: Đây là "bộ não" của app — quyết định customer có được edit hay không. Edit window bảo vệ merchant khỏi việc customer edit đơn đã ship. 100% đối thủ đều có time window — đây là table stakes.

Trông như thế nào: Settings page: slider chọn time window (30m, 1h, 2h, 6h, 12h, 24h, until fulfillment). Toggle on/off edit per order status. Backend: mọi request edit → check eligibility → allow/deny với message cụ thể.

1.4
Address Edit (Customer Self-Service)
ICE 630
Nó là gì: Cho phép customer tự sửa địa chỉ giao hàng sau khi đặt hàng. Gọi Shopify Order Edit API (graphQL beginEdit → setOrderShippingAddress → commitEdit). Validate address format, tính lại shipping rate nếu cần.

Để làm gì: 30% customer mắc lỗi địa chỉ khi checkout. Đây là use case #1 được merchant yêu cầu. 60% support tickets là về chỉnh sửa địa chỉ. Giải quyết vấn đề này = giảm 60% workload support.

Trông như thế nào: Customer vào Order Status Page → thấy nút "Edit Order" → click → form edit địa chỉ (prefilled data hiện tại) → sửa → Submit → thông báo thành công. Merchant nhận email notification về thay đổi.

1.5
Variant Swap (Customer Self-Service)
ICE 486
Nó là gì: Cho phép customer đổi variant sản phẩm (size, color, style) trong đơn hàng. Gọi Shopify Order Edit API: removeLineItem + addLineItem. Tự động tính chênh lệch giá, xử lý charge thêm hoặc refund.

Để làm gì: "Tôi chọn nhầm size" là lý do #2 khách hàng liên hệ support. Đổi variant giúp giảm return rate 20-25% (return do lỗi customer, không phải lỗi sản phẩm). Tất cả đối thủ đều có — đây là core feature.

Trông như thế nào: Customer click "Edit Order" → thấy danh sách line items → click "Change" bên cạnh item → popup chọn variant mới (size/color picker) → hiện chênh lệch giá (nếu có) → Confirm → Shopify tự charge/refund.

1.6
Quantity Adjustment
ICE 504
Nó là gì: Cho phép customer tăng/giảm số lượng sản phẩm trong đơn hàng. Check inventory availability khi tăng. Tự động tính lại tổng tiền, xử lý refund khi giảm, charge khi tăng.

Để làm gì: Customer thường muốn thêm sản phẩm (quên mua) hoặc bớt (mua nhiều quá). Điều chỉnh số lượng đơn giản hơn huỷ + đặt lại. Giảm return rate và tăng AOV (Average Order Value).

Trông như thế nào: Trong edit page: mỗi line item có nút +/- để điều chỉnh quantity. Hiện inventory còn lại. Hiện giá mới real-time. Confirm → update order qua Shopify API.

1.7
Order Cancellation + Inventory Restock
ICE 504
Nó là gì: Cho phép customer tự huỷ đơn hàng. Tự động restock inventory, xử lý refund (full hoặc partial). Cancel reason dropdown (changed mind, found better price, ordered by mistake, other). Merchant config: cho phép/không cho cancel, thời gian cho phép cancel.

Để làm gì: Customer muốn huỷ đơn nhưng phải liên hệ support = friction. Tự động cancel giảm workload, restock inventory ngay = không mất bán. Cancel reason data giúp merchant hiểu tại sao customer huỷ.

Trông như thế nào: Edit page có nút "Cancel Order" màu đỏ. Click → confirmation modal với dropdown reason → Confirm → order cancelled, inventory restocked, refund processed. Merchant nhận notification.

1.8
Auto Refund/Charge on Price Difference
ICE 360
Nó là gì: Xử lý tự động chênh lệch giá khi customer edit đơn. Nếu giá tăng: charge thêm qua Shopify payment. Nếu giá giảm: refund chênh lệch. Hiện rõ chênh lệch trước khi confirm. Support cả partial refund.

Để làm gì: Đây là pain point lớn nhất của đối thủ — nhiều app tính sai thuế, charge trùng, hoặc không refund. Làm đúng = trust. Làm sai = 1-star review + mất merchant. "It doubled the customer's charge" là review 1 sao phổ biến nhất.

Trông như thế nào: Khi customer edit → hiện bảng so sánh: "Before: $50 → After: $45 → Refund: $5". Nút Confirm hiện rõ số tiền. Backend: gọi Shopify Refund API hoặc Order Edit API để xử lý. Transaction log lưu mọi thay đổi.

Merchant Admin — Order List + Edit Modal
ICE 432

2.1
Email Notifications (Merchant + Customer)
ICE 567
Nó là gì: Hệ thống gửi email tự động: cho customer (xác nhận edit thành công, xác nhận cancel, refund processed), cho merchant (thông báo customer edit/cancel đơn, daily summary). Template responsive HTML.

Để làm gì: Customer edit xong mà không nhận email = không tin tưởng. Merchant không biết customer edit = mất kiểm soát. Email là communication channel thiết yếu — 100% đối thủ đều có.

Trông như thế nào: Customer edit address → nhận email "Your shipping address has been updated" với chi tiết cũ → mới. Merchant nhận email "Order #1234 was edited by customer" với summary thay đổi. Settings: toggle on/off từng loại email.
