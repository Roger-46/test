# Requirements Documentation

This folder contains all requirement specifications for Affily features.

## File Structure

- **template-user-story.md** - Template for new user stories
- **core-features.md** - Core affiliate features requirements
- **shopify-integration.md** - Shopify integration requirements
- **user-stories.md** - All user stories in one place

## How to Use

1. Copy `template-user-story.md` for new features
2. Fill in: User Story, Acceptance Criteria, Rationale, Dependencies
3. Set Priority based on MVP scope
4. Update status as you work

## Naming Convention

- Use kebab-case: `feature-name.md`
- Include date if time-sensitive: `feature-name-2026-03.md`
- Group by feature area: `affiliate-onboarding.md`, `commission-tracking.md`

## Process

1. **Draft** → Write requirement, ask BA for review
2. **Review** → BA reviews with team, gets clarification
3. **Ready** → Shared with dev team for estimation
4. **In Progress** → Dev team starts implementation
5. **Done** → Feature shipped, requirement archived
