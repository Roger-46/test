# Architecture Documentation

This folder contains technical architecture & design documents.

## Files

- **system-overview.md** - High-level system architecture
- **api-endpoints.md** - REST API specifications
- **database-schema.md** - Data model & schema
- **integrations.md** - External integrations (Shopify, etc.)

## When to Use

- **For BA:** Understand how requirements map to technical architecture
- **For Devs:** Reference for implementation details
- **For Planning:** Identify dependencies & constraints

## Key Diagrams

Architecture files should include:
1. **System Diagram** - How components interact
2. **Data Flow** - How data moves through system
3. **API Contracts** - Request/response formats
4. **Database Schema** - Entity relationships

## Template Structure

```markdown
# [Component Name]

## Overview
[Brief description]

## Architecture
[Diagram or ASCII representation]

## Key Components
- Component A
- Component B

## Data Flow
[Describe how data moves]

## Constraints
[Technical limitations]

## Future Improvements
[Potential enhancements]
```
