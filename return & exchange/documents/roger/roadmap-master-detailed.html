1.1
Return Portal (Customer-facing)
ICE 600
No la gi: Trang self-service cho customer tao return request. Customer nhap order number + email de lookup don hang, chon items muon return (multi-item, tung quantity), chon return reason tu dropdown (8+ predefined reasons), upload anh (toi 3 anh/item, 10MB max), va chon resolution (refund hoac store credit). Trang nay embed tren storefront qua Theme App Extension (Liquid block).

De lam gi: Day la core feature so 1 — khong co return portal = khong co app. Customer can tu phuc vu 24/7 ma khong can email merchant. 67% customers muon self-service return hon la lien he support. Portal giup giam 80% support tickets lien quan den returns.

Trong nhu the nao: Order Lookup: form nhap Order # + Email, validate va hien thi eligible items voi anh, ten, gia, quantity. Item Selection: checkbox tung item + quantity picker. Reason Selection: dropdown per item (Size too small/large, Damaged, Not as described, Changed mind, Wrong item, Better price elsewhere, Other). Photo Upload: drag-and-drop zone, preview thumbnails. Resolution Choice: radio buttons "Refund to original payment" hoac "Store Credit (Gift Card)". Confirmation: summary page voi return ID va next steps.

Refund & Store Credit Processing
ICE 450
No la gi: He thong xu ly refund va store credit khi merchant approve return. Refund: tao Shopify refund qua GraphQL API (full/partial amount, restocking fee deduction). Store Credit: tao Shopify Gift Card qua API, gui code cho customer qua email. Ho tro exchange bonus credit (khich le customer chon store credit thay vi refund).

De lam gi: Day la buoc cuoi cung cua return flow — customer muon tien lai hoac credit de mua tiep. Khong co processing = merchant phai lam thu cong tren Shopify Admin, defeat purpose cua app. Idempotent operations de tranh duplicate refunds.

Trong nhu the nao: Refund Modal: hien refund amount (pre-calculated), restocking fee deduction (neu bat), partial refund adjustment, confirm button. Store Credit Modal: amount + optional exchange bonus (VD: "+10% bonus = $55 thay vi $50"), confirm → tao gift card → gui email voi code. History: tab Refunds/Credits hien toan bo lich su xu ly. Safety: duplicate protection (khong refund 2 lan), amount validation (khong vuot tong don).

2A.9
Rate Limiting & Portal Security
ICE 567
No la gi: Bao ve return portal khoi abuse: rate limiting (max 5 return requests/email/ngay, max 10 lookups/IP/phut), CAPTCHA cho form submission, input sanitization, file upload validation (type, size, malware scan basic). Honeypot fields chong bot.

De lam gi: Return portal la public-facing endpoint — bat ky ai cung co the truy cap. Khong co rate limiting = bot spam hang ngan fake return requests. Khong co file validation = upload malware. Security phai co tu dau, khong the patch sau khi bi attack.

Trong nhu the nao: Rate Limiting: Redis-based counter per IP va per email. Hien "Too many requests, please try again later" khi vuot limit. CAPTCHA: Google reCAPTCHA v3 (invisible) cho form submission. File Upload: validate file type (jpg/png/gif/webp only), max 10MB, strip EXIF metadata. Input: sanitize tat ca user inputs, prevent XSS/injection.
